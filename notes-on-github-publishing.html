Notes on github publishing

Links on generating github pages:

mostly followed http://www.circuidipity.com/github-pages.html

Other usefull sites http://mathamy.com/migrating-to-github-pages-using-pelican.html

Project pages vs. User pages http://docs.getpelican.com/en/3.3.0/tips.html

http://magically.us/2013-02-03/creating-a-pelican-powered-site-on-github-pages.html

needed to change

SITEURL = 'http://elfnor.github.io/lookthinkmake'

in publishconf.py

process:

edit files in ~/lookthinkmake/content etc.

check it looks OK

source ~/virtualenvs/lookthinkmake/bin/activate
make devserver

point browser at localhost:8000

to publish

./develop_server.sh stop # if devserver running
source ~/virtualenvs/lookthinkmake/bin/activate #if not already in virtual env
cd lookthinkmake
git add content/
git commit -a -m "comment on changes"
git push -u origin master
make github

point browser at http://elfnor.github.io/lookthinkmake

to make a new git repository on github

In the directory on my machine: add a .gitignore file and a readme.md file to the directory (both optional)

git init
git add .
git status
git commit -a -m "useful comment"
git log

This has created the local repository. Go to github https://www.github.com/elfnor login and make a new repository there.

In the directory on my machine:

git remote add origin https://github.com/elfnor/newreponame.git
git remote -v
git push -u origin master

This labels the repo on github as origin. (could be any name) and pushes the repo in the local directory to github.

The github repo can be deleted from github if you mess up.

Project Pages

after creating gallery

with structure

/home/elfnor/Documents/GIMP

To make this into project pages, that is viewable with a browser:

https://help.github.com/articles/creating-project-pages-manually

$ git clone https://github.com/user/repository.git

change user and repository to actual names git clone makes a copy of the repo on github on my computer in the sub folder repository. ignore this if the repository already exists locally

$ cd repository

$ git checkout --orphan gh-pages

this makes a new branch of that repository called gh-pages which is an "orphan", not sure what this means, maybe that its empty?

git checkout --orphan newbranch Create a new orphan branch, named , and switch to it. The first commit made on this new branch will have no parents and it will be the root of a new history totally disconnected from all the other branches and commits.

git rm -rf . Remove all files from the old working tree. This empties the local directory

copy the files you want on the project pages back into the local directory

git add . add the content to the index for gh-pages branch

git commit -a -m "First pages commit" commit the content to the gh-pages branch

git push origin gh-pages

now the github branch gh-pages contains the html files for the gallery

after this the master branch still contains all the files I put there by accident? need to remove locally and push it back to github?

Maybe ????

git checkout master

delete all the files

git add -u .

git commit -a -m "delete master"

git push origin master

Yes this worked

then get the files back in the local directory linked to gh-pages branch

git checkout gh-pages
